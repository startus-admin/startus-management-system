# AI機能搭載プラン - STARTUS Management System

作成日: 2026年3月10日

---

## 1. 概要

STARTUS管理システムにAI（人工知能）機能を搭載し、スタッフの業務効率化を図るプラン。
AIには Anthropic社の「Claude」APIを使用する。

---

## 2. 搭載するAI機能（3つ）

### 機能① AIチャットアシスタント

スタッフが日本語で質問すると、システムのデータベースを検索して回答するAIアシスタント。

**使い方の例:**
- 「今月の体験申し込みは何件？」→ 申請データを検索して件数を回答
- 「在籍会員数は？」→ 会員テーブルからカウントして回答
- 「教室ごとの人数を教えて」→ 教室別の在籍数を表形式で回答
- 「未処理の申請はある？」→ ステータスが未対応の申請を一覧表示

**画面イメージ:**
- 画面右下にAIアイコンのボタンを配置
- クリックするとチャットパネルが開く（既存のチャット機能と似た操作感）

---

### 機能② データ分析・AIレポート

AIがデータを自動集計・分析し、レポートとして表示する機能。

**レポートの種類:**

| レポート名 | 内容 |
|-----------|------|
| 会員動向分析 | 入退会の推移、成長率、減少傾向の検出 |
| 月謝状況レポート | 未納者の早期検出、徴収率の推移 |
| 教室稼働率 | 教室ごとの在籍数とキャパシティの比較 |
| 体験転換レポート | 体験→入会の転換率分析 |
| 月次サマリー | 今月の主要指標まとめと先月比較 |

**画面イメージ:**
- サイドバーに「AIレポート」タブを追加
- カード形式で各レポートを表示、ボタンを押すとAIが分析結果を生成

---

### 機能③ 業務自動化支援

既存の画面にAIボタンを追加し、定型業務をAIが補助する機能。

| 機能 | 表示場所 | 内容 |
|------|---------|------|
| AIチェック | 申請詳細画面 | 申請フォームの不備・不足を自動検出 |
| フォローアップ文面生成 | 体験詳細画面 | 体験後のメール文面をAIが自動作成 |
| AIおすすめ | ダッシュボード | 「未対応申請が2件あります」等の提案表示 |

---

## 3. 技術構成

```
[スタッフのブラウザ]
    ↓ 質問を送信（暗号化通信）
[Vercelサーバー]  ← 既存のサーバー（追加コストなし）
    ↓ データベースから必要な情報を取得
    ↓ AIに質問+データを送信
[Anthropic Claude API]  ← AI処理
    ↓ 回答を生成
[Vercelサーバー]
    ↓ 回答を返送
[スタッフのブラウザに表示]
```

**ポイント:**
- AIのAPIキーはサーバー側に保管（ブラウザからは見えない）
- 既存のシステム構成（Vercel + Supabase）をそのまま活用
- 新たなサーバー契約は不要

## 6. 個人情報の取り扱い

### 会員データに含まれる個人情報

| 分類 | フィールド |
|------|-----------|
| 特に機密（要配慮個人情報） | 障がい情報（disability_info） |
| 高リスク | 住所、電話番号、メールアドレス、生年月日 |
| 未成年関連 | 保護者名、学校名、学年 |
| 識別情報 | 氏名、フリガナ、写真 |

### Anthropic社のデータポリシー（API利用時）

| 項目 | 内容 |
|------|------|
| モデル学習への使用 | **されない**（APIデータは学習に不使用） |
| データ保持期間 | 安全監視のため最大30日（Enterprise契約で0日も可能） |
| 暗号化 | 転送中・保存中ともに暗号化 |
| 第三者への販売 | **なし** |

### 実装で行う個人情報保護対策

| 対策 | 内容 |
|------|------|
| データ最小化 | AIに送るデータを必要最小限に制限（全項目は送らない） |
| PIIマスキング | 住所→都道府県のみ、電話→090-****-1234 のように加工 |
| 要配慮情報の除外 | 障がい情報はAIに一切送信しない |
| 集計データ優先 | レポート機能では個人データでなく統計値のみ使用 |
| 認証必須 | ログイン済みスタッフのみAI機能を利用可能 |

### 法令関連

- 日本の**個人情報保護法（APPI）**の対象
- APIへのデータ送信は「第三者提供」に該当する可能性あり
- 対策: データ最小化・マスキングにより、個人を特定できない形で送信
- 必要に応じて: プライバシーポリシーの更新、利用者への通知
